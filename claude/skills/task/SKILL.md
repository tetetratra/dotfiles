## 使用例

```
/task （URLなど）のチケットをお願いします
```

## 目的
与えられたタスクを、自らロードマップを引きつつ進めます。

## 手順

1. まず、作業における中間成果物（目的・背景・方針案・設計 などの文章）をどこに保存すべきかをユーザーに質問してください
  - 選択肢としては下記などがあります
    - チケット上（GitHub Issues、Notion など）
    - ローカルファイル（`tmp/{タスク名}/`配下などにmarkdownファイルを保存）
    - 保存なし（会話上で完結）

2. 次に、自律レベルの説明をしてください。そして、今回のタスクではどの自律レベルで動いてほしいかを3段階で答えるよう、質問してください（既に指示されていた場合はスキップ）
  - 以降の作業は、指示された自律レベルの行動指針に従って作業をしてください

3. `dev-step--plan-roadmap` スキルを使い、タスクのロードマップを引いてください。
  - タスクの内容によってはロードマップを引くために必要な情報が、ロードマップの中のステップを進める過程でしか得られないことがあるため、あえて最初から完全なロードマップは引かないような指示としています

4. 引いたロードマップに従って、順々に作業ステップを進めてください

5. 引いたロードマップが完了した場合、改めて `dev-step--plan-roadmap` スキルを使い、現在までのロードマップに続くかたちで更なるロードマップを引いてください
  - 以降、同様のサイクルで作業を進めてください
  - 引いたロードマップが完了し、かつ最初に与えられたタスクも全て完了した場合、作業完了です。ユーザーに完了報告をしてください

## 自律レベル

AIにどのくらい自律的に動いてもらいたいかを「自律レベル」とし、以下の3段階で定義します。
自律レベルに応じて、以下の行動指針に従ってください。

- レベル1
  - 意思決定が必要な状況では常にユーザーに確認をとる
  - 作業ステップが完了するごとに、常にユーザーに報告・確認を求める
- レベル2
  - 意思決定が必要な状況のうち、AIが自信をもって判断できない状況に限り、ユーザーに確認をとる
  - 作業ステップが完了した際のユーザーへの報告・確認はせずに次の作業ステップに進むこと
- レベル3
  - すべての意思決定をAIが行う
  - 作業ステップが完了した際のユーザーへの報告・確認はせずに次の作業ステップに進むこと

例外的に、以下の状況に陥った場合に限り作業の手を止めて、ユーザーからの指示を待って下さい

- 危険を伴う操作（外部リソースの操作・復元不可能な変更）を実行しようとする場合
- 作業がロードマップ通りに進まず、手詰まりになった場合
- 重大なインシデントを起こしたことに気づいた場合

## 作業を進める際の指針
- 推奨スキルが指定されていた場合、基本的にはそのスキルを使ってください（他のスキルと併用してもOK）
- 使用するスキル次第では「ユーザーに質問を投げてください」という指示が書かれている場合があると思います。
  そのような場合であっても、自律レベルに記載の行動指針を優先し、自律レベル次第ではあなた(AI)自身で判断してください

## ユーザーにレスポンス返す際のフォーマット

レスポンスの冒頭に以下のフォーマットで、現在のロードマップの状況を軽く提示してから、本題のレスポンスを書いてください。

```
## 現在のロードマップの状況
- ✅️ {完了したステップ名}
- ✅️ {完了したステップ名}
- ...
- ⏳ {現在取り組んでいるステップ名}
```
