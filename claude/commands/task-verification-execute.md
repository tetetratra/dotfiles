検証計画で洗い出した確認項目を実際に実行し、結果を記録します。

## 目的

- 検証計画で作成した確認項目を1つずつ実施する
- 実施した項目にチェックを入れ、進捗を明確にする
- 問題があれば記録し、必要に応じて修正する
- 確認結果を記録し、PR 作成時に活用できるようにする

## 入力で確認すること

- 対象タスクのリンクと概要
- 検証計画で作成した確認項目リスト
- 検証環境（開発環境、QA環境）
- テストデータの準備状況
- 情報が不足している場合は、着手前に質問を箇条書きする

## 検証実行における重要な指示

### 検証対象環境について
- **開発環境でのみ検証を実施する**
- QA環境での検証は実施しない（人間が別途行うため）
- 検証計画にQA環境での確認項目が含まれていても、開発環境のみで実施する

### 環境の準備と操作
- ローカルでRailsサーバーを起動する（`rails server` など）
- フロントエンドのビルドやサーバーを起動する（`npm run dev` など）
- 必要なサービス（データベース、Redis等）が起動していることを確認する
- サーバー立ち上げやビルドの手順がわからない場合は、リポジトリの CLAUDE.md や README.md 等のドキュメントを参照する

### ブラウザ操作による検証
- Playwrightを使用してブラウザを起動し、実際のユーザー操作をシミュレートする
- 画面の表示、ボタンのクリック、フォーム入力などを実際に操作して確認する
- 必要に応じてスクリーンショットを取得する

### 検証完了後の後片付け
- すべての確認項目の検証が完了したら、起動したサーバーやプロセスを停止する
- 確認項目ごとに起動・停止を繰り返すのではなく、一連の検証が終わってから停止する

## 出力物

このコマンドでは、以下の3つのカテゴリで出力を行います：

- **返答**: 作業完了の報告や調査結果の要約など、フロー型の出力（会話として返す）
- **成果物**: ドキュメントや整理結果など、ストック型の出力（後から参照できる形で返す）
  - 成果物の保存先について指示がある場合は、指定された箇所に書き込む
  - 指示がない場合は、通常のレスポンスとして返答の後に続けて出力する
- **作業**: システムへの変更や実行など、副作用を伴う作業

本コマンドの出力：

**返答**: 各確認項目の実施報告、問題発生時の報告、検証完了の報告

**成果物**: チェック済みの確認項目リストをレスポンスとして出力する

**作業**: サーバー起動、ブラウザ操作、検証実施、問題発見時の修正、サーバー停止

## 手順

### 1. 検証環境の準備

- 開発環境とQA環境を準備（確認項目に応じてどちらか、または両方）
- 必要なテストデータを準備
- 実装した機能が動作する状態にする

### 2. 確認項目の実施

検証計画で作成した確認項目を順に実施する：

- 確認項目の内容を理解する
- 実際に操作や確認を行う
- 期待通りの動作をするか確認する
- 実施した項目にチェックを入れる（`- [x]`）

### 3. 結果の記録

各確認項目について：

- 正常に動作した場合: チェックを入れ、必要に応じて結果を簡潔に記録
- 問題があった場合: 問題内容を記録し、修正が必要か判断
- スクリーンショットやログが有用な場合: その旨を記載

### 4. 問題への対応

問題が見つかった場合：

- 問題の内容を明確に記録
- 修正が必要な場合は、実装に戻って修正
- 修正後、再度確認を実施

### 5. 確認結果の整理

- すべての確認項目が実施されたか確認
- 未実施の項目があれば、理由を記載
- 全体としての確認結果をまとめる

## 成果物例

既に動作確認項目がある想定なので、それのチェックボックスを付けたものを出力すること。

```
## 動作確認項目

開発環境で、以下を確認した
- [x] 管理画面での日時入力が正しく動作することを確認
  - Playwrightでブラウザを起動し、管理画面にアクセス
  - datetime-local フィールドで 2025-12-09 14:30 を入力し、保存ボタンをクリック
  - データベースを確認し、正しく保存されていることを確認
- [x] 既存データの表示が正しく行われることを確認
  - 既存のテストデータが一覧画面に正しく表示されることを確認

QA環境で、以下を確認した
- [ ] ...
- ...
```

## チェックポイント

- [ ] 開発環境でのみ検証を実施したか（QA環境では実施していないか）
- [ ] 必要なサーバーやサービスを起動したか
- [ ] Playwrightを使用してブラウザ操作で確認したか
- [ ] すべての確認項目を実施したか
- [ ] 実施した項目にチェックが入っているか（`- [x]`）
- [ ] 確認結果が記録されているか
- [ ] 問題が見つかった場合、その内容が記録されているか
- [ ] 必要に応じて修正を行ったか
- [ ] スクリーンショットやログなど、証跡が必要な場合は記録したか
- [ ] 全体としての確認結果がまとめられているか
- [ ] 検証完了後、起動したサーバーやプロセスを停止したか
