検証計画で洗い出した確認項目を実際に実行し、結果を記録します。

## 目的

- 検証計画で作成した確認項目を1つずつ実施する
- 実施した項目にチェックを入れ、進捗を明確にする
- 問題があれば記録し、必要に応じて修正する
- 確認結果を記録し、PR 作成時に活用できるようにする

## 入力で確認すること

- 対象タスクのリンクと概要
- 検証計画で作成した確認項目リスト
- 検証環境（開発環境、QA環境）
- テストデータの準備状況
- 情報が不足している場合は、着手前に質問を箇条書きする

## 出力の要件

- レスポンスとして直接返す（ドキュメントファイルは作成しない）
- 検証計画の確認項目をベースに、実施結果を記録する
- 実施した項目はチェックを入れる（`- [x]`）
- 必要に応じて確認結果の詳細や補足情報を記載する
- スクリーンショットやログが必要な場合は、その旨を記載する

## 手順

### 1. 検証環境の準備

- 開発環境とQA環境を準備（確認項目に応じてどちらか、または両方）
- 必要なテストデータを準備
- 実装した機能が動作する状態にする

### 2. 確認項目の実施

検証計画で作成した確認項目を順に実施する：

- 確認項目の内容を理解する
- 実際に操作や確認を行う
- 期待通りの動作をするか確認する
- 実施した項目にチェックを入れる（`- [x]`）

### 3. 結果の記録

各確認項目について：

- 正常に動作した場合: チェックを入れ、必要に応じて結果を簡潔に記録
- 問題があった場合: 問題内容を記録し、修正が必要か判断
- スクリーンショットやログが有用な場合: その旨を記載

### 4. 問題への対応

問題が見つかった場合：

- 問題の内容を明確に記録
- 修正が必要な場合は、実装に戻って修正
- 修正後、再度確認を実施

### 5. 確認結果の整理

- すべての確認項目が実施されたか確認
- 未実施の項目があれば、理由を記載
- 全体としての確認結果をまとめる

## 出力例

```
## 動作確認結果

開発環境で、以下を確認した
- [x] 管理画面での日時入力が正しく動作することを確認
  - datetime-local フィールドで 2025-12-09 14:30 を入力し、正しく保存された
- [x] 既存データの表示が正しく行われることを確認
  - 問題なし
- [x] 大量データでのパフォーマンスに影響がないことを確認
  - 1000件のデータで一覧表示時間は約200ms。問題なし

QA環境で、以下を確認した
- [x] 管理画面での日時入力が正しく動作することを確認
  - 問題なし
- [x] 既存データの表示が正しく行われることを確認
  - 問題なし
```

## チェックポイント

- [ ] 検証環境が正しく準備されているか
- [ ] すべての確認項目を実施したか
- [ ] 実施した項目にチェックが入っているか（`- [x]`）
- [ ] 確認結果が記録されているか
- [ ] 問題が見つかった場合、その内容が記録されているか
- [ ] 必要に応じて修正を行ったか
- [ ] スクリーンショットやログなど、証跡が必要な場合は記録したか
- [ ] 全体としての確認結果がまとめられているか
- [ ] 未実施の項目がある場合、その理由が記載されているか
