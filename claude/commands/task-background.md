タスクのビジネス的背景・目的を体系的に定義し、チケットに記載します。

## 使用例

```
/task-background Notionのこのチケット https://notion.so/xxxxx に対して目的・背景セクションを作成してください
```

## 手順

1. **タスクの表面的要求の把握**
   - タスクの記載内容から、何を作ろう・直そうとしているかを整理する
   - 「このタスクは何をするものか？」を明確にする

2. **根本的なビジネス課題の探索**
   - 「なぜその対応が必要なのか？」を深掘りする
   - 機能追加や修正の裏にあるビジネス上の課題や機会を探る
   - ユーザーや事業に対してどのような価値を提供するかを考える

3. **背景情報の収集**
   - タスクに関連するドキュメント、過去の議論、関連チケットを確認
   - 類似の課題や過去の対応事例を調査
   - ステークホルダーからの要求や制約事項を整理
   - 必要に応じて関連するシステムの現状を確認

4. **ビジネス価値の連鎖整理**
   以下の3層でビジネス価値を段階的に整理し、それぞれを文章として明確に表現する：
   ```
   根本価値: ユーザーや事業に対する最終的な価値提供（なぜこの取り組みが重要か）
   ↓
   中間価値: 機能やプロセスの改善による効果（どのような改善で価値を実現するか）
   ↓
   直接的作業: 今回のタスクで実際に行う開発・対応内容（具体的に何を作るか）
   ```

5. **目的・背景セクション作成**
   - 内容
       - 現状の課題やビジネス機会を説明
       - なぜ今この対応が必要かの論理的つながりを示す
       - 期待される効果やユーザーへの価値を明確にする
       - 「◯◯を実現する必要があり、本タスクで対応します」のように、タスクの目的を明確に述べる
   - 記法
       - 基本的に見出しや太字は避け、文章で書く（バッククォートによる囲みは使用可）
       - 数文程度で、最低限の事項のみを簡潔に記述する
       - 文ごとに改行を入れ、積極的に段落を区切り、空行を入れる
       - 関連チケットや資料に対する言及は、初回登場時のみリンクとして記載する (明らかに分かる場合を除く)

例
```
現状の課題として、メンテナンス時のアラート表示に[hogehoge](https://example.com/hoge)を経由する必要があり、開発チームのサポートが必要だった。

本タスク(or PR or チケット)では、アラート機能をhogehogeではなくリポジトリのコード側で処理するようにし、
Bizチームが自立的にfugafugaを行える体制を構築する。
```

6. **指定された場所への反映**
   - 指示された方法でタスクの目的・背景セクションを実際に記載する
   - `## 目的・背景` セクションを追加
   - 既存の構造を維持しながら新しいセクションを挿入

7. **要求定義の妥当性確認**
   - ビジネス価値が明確に表現されているか
   - 実装者が「なぜ」を理解できる内容か

## 実行例

```bash
# 1. タスクの表面的内容確認
# 指定されたタスク（Notion、GitHub Issue等）の内容を確認

# 2. 関連情報収集
# 関連チケット、過去の議論、要求仕様を調査
# 必要に応じて類似機能や競合他社の事例も確認

# 3. ビジネス価値連鎖の整理
# 根本価値 → 中間価値 → 直接的作業の流れを文章化

# 4. 目的・背景セクション作成
# - 現状の課題・機会
# - 対応の必要性
# - 期待効果
# - タスクの位置づけ

# 5. 指定場所への反映
# タスク管理ツールやドキュメントに ## 目的・背景 セクションを追加
```

## チェックポイント

- [ ] ビジネス上の「なぜ」が明確になっているか
- [ ] 技術的作業とビジネス価値のつながりが説明されているか
- [ ] ユーザーや事業への価値が具体的に示されているか
- [ ] 実装者が目的を理解できる情報が含まれているか
- [ ] 今回の対応の優先度・重要性が伝わるか
- [ ] ですます調の文章構成になっているか
- [ ] 文章が3〜5文程度で簡潔に記述されているか
- [ ] 文ごとに改行され、段落の区切りには空行が入っているか
- [ ] 関連チケットや資料の言及がリンクとして記載されているか
- [ ] `## 目的・背景` セクションが指定場所に追加されているか
- [ ] ビジネス要求が技術要求に適切に変換できる内容か
