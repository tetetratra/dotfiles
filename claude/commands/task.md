# 概要
このドキュメントは、タスクの明確化と実装に関する標準的な手順を示しています。

まず、与えられたnotionページやmdファイルなどの形式で提供されたタスクのチケットに、`## 手順`セクションの記載が無い場合、この資料の`## 手順`セクションをそのまま転記してください。

その後、メタ手順に従って、それぞれ1〜5までの各手順について、順番に実行してください。

## メタ手順

各手順は、以下の手順で実行してください。

- チケットに、下記具体例のフォーマットで空のセクションを追加する

```
## 1. コンテキスト収集・要件定義
### TODO

### アウトプット

```

- TODOセクションに、その手順で実行する具体的なタスクをリストアップする
  - 各タスクについて、何を成果物とするかも明示すること
    - コードベースを理解する行為・実装する行為などの、文書化できない作業についてはアウトプット欄に成果物の記載不要

具体例
```
### TODO
- 背景・経緯の理解：関連Notionページの確認
- 既存の類似実装の調査
  - アウトプット：類似実装がある場合、そのリスト
- ...
```

- TODOセクションについてユーザーの確認を求め、OKが出るまで手戻りを行う

- TODOを順次実行する
  - アウトプット欄
    - 記入タイミング：TODOをひとつ実行するごと
    - 内容：重要な情報のみを簡潔にまとめること。当たり前なことや、類推できることは省略すること
    - フォーマット：箇条書きで書き(小見出しは禁止)、100行以内に収めること
  - 完了したTODOにはチェックを入れること

具体例
```
### アウトプット
- 背景・経緯: mm/ddに、xxのためにyyがエラーで失敗した
  - 原因: ...
  - 根本原因: ... 
- zzzの構造
  - ...
- ...
```

- 実行結果についてユーザーの確認を求め、OKが出るまで手戻りを行う

- 次の手順へ進む

## 手順

1. コンテキスト収集・要求定義
  - 背景・経緯の理解
  - コードベースの探索
  - 既存の類似実装の調査
  - 要求定義の明確化

2. 設計
  - 概要方針案の列挙
  - 概要方針の決定
  - 設計案の作成
  - 設計を確定

3. 実装
  - 骨組み部分のみを実装
  - メインのコードを実装
  - 枝葉のコードを実装
  - テストコードを実装

4. 検証・フィードバック
  - 動作確認の実行
  - 問題の特定と修正
  - レビュー依頼

5. (optional) 報告・ドキュメント化
  - ドキュメントの作成・更新
  - 作業結果のまとめを作成
