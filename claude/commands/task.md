# 概要
この親エージェントは、タスクの明確化と実装に関する標準的な手順を管理し、各専門エージェントと協調して作業を進めます。

各手順について、対応する子エージェント（task-XXX.md）を順番に呼び出して作業を進めます。

親エージェント（このファイル）が全体の進行を管理し、各手順で対応する子エージェントを呼び出します。

**[重要] 各子エージェントは必ずサブエージェントとして動かし、あなた自身は直接作業を行わないでください。**
 
## 基本フロー

1. **手順実行**: 以下の順序で子エージェントを呼び出し、段階的に作業を進める（直列で実行すること）
   - `/task-context` (コンテキスト収集・要求定義)
   - `/task-design` (設計)
   - `/task-implementation` (実装)
   - `/task-release` (リリース)

2. **進捗管理**: 各子エージェント完了後、次の段階に進む前にユーザー確認を求める

## 子エージェントへの指示事項

各子エージェントには、以下のルールに従って動作するように指示してください。

1. チケットに該当手順の空のセクションを追加
  - セクションには「### タスク」「### アウトプット」「### 引き継ぎ事項」欄のみを含むこと
  - セクション形式例
    ```markdown
    ## 1. コンテキスト収集・要件定義
    ### タスク
    ### アウトプット
    ### 引き継ぎ事項
    ```

2. 「チケットの全体(特に直近の作業)」と「子エージェント自身に与えられた指示」を読み、理解する

3. タスクセクションに具体的なタスクをリストアップ
  - タスクリスト作成時は適宜以下のタグを付与すること。タグが無いタスクが存在してよい
    - **[Check]** 作業完了後に手を止め、ユーザーに作業結果の確認を得るときに付けるタグ
    - **[User]** ユーザーに作業を依頼するときに付けるタグ
    - **[Output]** 完了後にアウトプット欄に記録を残す作業に付けるタグ
    - **[Impl]** コードベースに直接影響を与える実装タスクに付与するタグ
  - タスク形式例
    ```markdown
    ### タスク
    - [ ] 背景・経緯の理解：関連Notionページ/ドキュメントの確認
    - [ ] タスクの目的と制約の明確化
    - [ ] ...
    - [ ] [Output] コードベースの構造調査・調査結果まとめ
    - [ ] [Output, Check] 要求定義を明確化してまとめ、ユーザーに確認を求める
    ```

4. タスクのリストアップ後、ユーザーにタスクリストの承認を求める

5. タスクを順次実行
  - 付与したタグ通りに動作すること
  - アウトプット欄では装飾を避けて(小見出し・太字は禁止)、箇条書き(ネスト可)で簡潔に記録すること
  - アウトプット例
    ```markdown
    - 目的：ユーザー登録フォームのリニューアル
    - 制約：既存のデザインシステムに準拠、レスポンシブ必須
    - 類似実装：ログインフォーム(src/components/LoginForm.tsx)を参考にできる
    - ...
    ```

6. 引き継ぎ事項欄に、次の子エージェントが効率的に作業できるよう必要な情報を記載する
  - 箇条書きで簡潔に記載すること
  - markdownであれば `<details>`, `<summary>` タグを使い、notionであれば折りたたみ機能を使い、折りたたむことができる形式で記載すること
  - アプトプット欄に記入済みの内容に関しては記載しないこと

## 子エージェントへ指示を出す上で注意すること

この親エージェントが子エージェント(サブエージェント)に指示を出す際、以下の点に注意してください。

- 指示通りの形式で成果物を出力したか確認すること
- 完了したタスクは、そのチェックボックスがチェックされていることを確認すること
