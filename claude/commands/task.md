# 概要
この親エージェントは、タスクの明確化と実装に関する標準的な手順を管理し、各専門エージェントと協調して作業を進めます。

各手順について、対応する子エージェント（task-XXX.md）を順番に呼び出して作業を進めます。

親エージェント（このファイル）が全体の進行を管理し、各手順で対応する子エージェントを呼び出します。

## 基本フロー

1. **手順実行**: 以下の順序で子エージェントを呼び出し、段階的に作業を進める（直列で実行すること）
   - `/task-context` (コンテキスト収集・要求定義)
   - `/task-design` (設計)
   - `/task-implementation` (実装)
   - `/task-release` (リリース)

2. **進捗管理**: 各子エージェント完了後、次の段階に進む前にユーザー確認を求める

## 子エージェントへの指示事項

各子エージェントは以下のルールに従って動作します：

1. チケットに該当手順の空のセクションを追加
  - セクションには「### タスク」「### アウトプット」欄のみを含むこと
  - セクション形式例
    ```markdown
    ## 1. コンテキスト収集・要件定義
    ### タスク
    ### アウトプット
    ```

2. タスクセクションに具体的なタスクをリストアップ
  - タスクリスト作成時は適宜以下のタグを付与すること。タグが無いタスクが存在してよい
    - **[Check]** 作業完了後に手を止め、ユーザーに作業結果の確認を得るときに付けるタグ
    - **[User]** ユーザーに作業を依頼するときに付けるタグ
    - **[Output]** 完了後にアウトプット欄に記録を残す作業に付けるタグ
    - **[Impl]** コードベースに直接影響を与える実装タスクに付与するタグ
  - タスク形式例
    ```markdown
    ### タスク
    - [ ] 背景・経緯の理解：関連Notionページ/ドキュメントの確認
    - [ ] タスクの目的と制約の明確化
    - [ ] ...
    - [ ] コードベースの構造調査
    - [ ] [Output] コードベースの概要をまとめる
    - [ ] 要求定義の検討
    - [ ] [Check] 要求定義をユーザーに確認する
    - [ ] [Output] 要求定義の明確化してまとめる
    ```

3. タスクのリストアップ後、ユーザーにタスクリストの承認を求める

4. タスクを順次実行
  - 付与したタグ通りに動作すること
  - アウトプット欄では装飾を避けて(小見出し・太字は禁止)、箇条書き(ネスト可)で簡潔に記録すること
  - アウトプット例
    ```markdown
    - 目的：ユーザー登録フォームのリニューアル
    - 制約：既存のデザインシステムに準拠、レスポンシブ必須
    - 類似実装：ログインフォーム(src/components/LoginForm.tsx)を参考にできる
    - ...
    ```
