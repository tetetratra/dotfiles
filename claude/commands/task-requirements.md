$ARGUMENTS

要件定義フェーズで、機能要件・非機能要件・スコープ・受け入れ条件を整理し、合意可能な形にまとめます。

## 前提

- 背景・目的、関連システム/技術の理解、必要な調査は前段で完了している想定
- それらの再整理は行わず、要件の確定に集中する

## 出力物

要件定義の整理結果（機能要件/非機能要件/スコープ/受け入れ条件/制約・依存/未確定事項）。

## 入力で確認すること

- 対象タスクのリンクと概要
- 既に決まっている前提・制約
- 関係者の期待や優先度
- 不明点があれば、着手前に質問を箇条書きする

## 手順

1. 機能要件を列挙し、優先度（Must/Should/Could）を付ける
2. 非機能要件（性能、可用性、セキュリティ、運用など）を整理する
3. スコープ（やる/やらない）を明確にする
4. 受け入れ条件を定義し、完了の判断基準を揃える
5. 前提・制約・依存関係を明文化する
6. 未確定事項を洗い出し、質問/保留/仮決めに分類する

```
- ...

### 前提・制約・依存関係
- ...

### 未確定事項
- 質問: ...
- 保留: ...
- 仮決め: ...
```

## チェックポイント

- [ ] 機能要件に優先度が付いているか
- [ ] 非機能要件が抜けていないか
- [ ] スコープ外が明示されているか
- [ ] 受け入れ条件が具体的か
- [ ] 制約・依存関係が明文化されているか
- [ ] 未確定事項が整理されているか
