現在のタスクの進捗状況を確認し、次に実施すべき作業を明確化します。

注意：このコマンドはドキュメントを作成せず、現状の把握と今後の作業の提案をレスポンスとして返すものです。

## 目的

- 現在までに完了した作業を整理する
- 残りの作業を洗い出し、優先順位を明確にする
- 次に着手すべき作業を提案する
- タスク全体の進捗状況を可視化する

## 入力で確認すること

- 対象タスクのリンクと概要
- これまでに作成したドキュメント（設計書、実装計画等）
- 既存のコードやコミット履歴
- タスクの目標と達成条件
- 情報が不足している場合は、着手前に質問を箇条書きする

## 出力の要件

- レスポンスとして直接返す（ドキュメントファイルは作成しない）
- 完了した作業と残り見込み作業を明確に分ける
- 作業の粒度は 1スキルあたり1作業程度
- 次に着手すべき作業を具体的に提案する
- 必要に応じて注意点や懸念事項を記載する
- 利用可能なスキルは起動時に提供されているため、それを参照して作業を提案する

## 手順

### 1. 現状の把握

以下を確認して、どこまで作業が進んでいるかを把握:

- タスクの目標と達成条件
- 作成済みのドキュメント（背景情報、解決方針、前提条件、技術アーキテクチャ、実装計画等）
- 実装の進捗状況
  - コミット履歴
  - マイルストーンの達成状況
  - テストの状況
- レビューの状況（PR作成済みか、レビュー待ちか等）
- 利用可能なスキル一覧を確認し、どのような作業が可能かを把握する

### 2. 完了した作業の整理

以下の観点で完了した作業を列挙:

- どの作業が完了しているか（背景情報の整理、解決方針の検討、実装計画の作成、実装の実行等）
- ドキュメントの作成状況
- コードの実装状況
- テストやレビューの状況

### 3. 残り作業の洗い出し

以下の観点で残り作業を洗い出す:

- 未着手の作業はあるか
- 実装計画がある場合、未完了のステップやマイルストーンは何か
- テストやレビューで残っている作業は何か
- ドキュメントの更新が必要か

### 4. 優先順位の判断

以下の順序で優先順位を判断:

1. 準備作業が未完了の場合は、それを優先
   - 背景情報の整理、解決方針の検討、前提条件の明確化、技術設計、実装計画の作成等
2. 実装計画がある場合は、実装の継続
   - 次のステップやマイルストーンに進む
3. 実装が完了している場合は、レビューやリリース準備
   - PR作成、レビュー対応、ドキュメント更新等

### 5. 次の作業の提案

- 次に着手すべき作業を具体的に提案する
- 利用可能なスキルの中から適切なものを選んで提案する
- 作業に必要な情報や準備事項があれば記載する

## 出力例

```
**完了した作業**
- 背景情報の整理
- 解決方針の検討
- 前提条件・制約・達成条件の明確化
- 技術アーキテクチャの設計

**残り見込み作業**
- 実装計画の作成
- 実装の実行
  - マイルストーン M1 まで
  - マイルストーン M2 まで
  - マイルストーン M3（最終確認）まで
- PR作成とレビュー対応
- ドキュメントの最終更新
```
