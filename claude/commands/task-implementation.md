$1

task-architecture.md で設計された内容に従って実装を行います。

**注意**
このコマンドは設計された技術仕様をそのまま実装するものです。
設計の変更や追加の検討は行わず、決定された仕様に従って実装することが目的です。

## 使用例

```
/task-implementation Notionのこのチケット https://notion.so/xxxxx に対して実装してください
```

## 手順

### 1. 設計内容の確認
- [ ] task-architecture.md の設計内容を確認
- [ ] 実装対象のファイル・クラス・メソッドを特定
- [ ] データベース変更が必要な場合はマイグレーション内容を確認

### 2. レビュー指針の確認
- [ ] レビュー指針ガイドラインに従います。docs/ 配下等でgrepし、ある場合は参照してください

### 3. 実装

**コア機能とサポート機能の定義:**
- **コア機能**: app/{model,view,controller} 配下にあるようなクラス（ビジネスロジックの中核）
- **サポート機能**: それ以外（マイグレーション、設定ファイル、ヘルパー、ライブラリ、バックグラウンドジョブ等）

#### 3.0 Phase 0: ブランチ作成
- [ ] ブランチが切られていない場合、新規ブランチをdevelopブランチから切り作成する

#### 3.1 Phase 1: コア機能の骨組み作成
- [ ] コア機能に必要なクラス・ファイルのガワを作成
- [ ] コア機能のpublicメソッドについて、中身は空でメソッドを定義
- [ ] 骨組み作成完了でコミット

#### 3.2 Phase 2: コア機能の実装
- [ ] コア機能のメソッドを一つずつ実装
- [ ] 実装したメソッドのテストを作成・実行
- [ ] 各メソッドの実装完了ごとにコミット

#### 3.3 Phase 3: サポート機能の実装
- [ ] サポート機能の骨組み（クラス・空メソッド）を作成
- [ ] サポート機能のメソッドを一つずつ実装
- [ ] 実装したメソッドのテストを作成・実行
- [ ] 各メソッドの実装完了ごとにコミット

#### 3.4 Phase 4: 統合テスト
- [ ] 全体の動作を確認する統合テストを作成・実行
- [ ] 統合テスト完了でコミット

### 4. 実装完了確認
- [ ] 設計で定義された全ての項目が実装されているか
- [ ] テストが全て通過しているか

## 実行例

コミットメッセージは適宜調整してください。

```bash
# 1. 設計確認
# task-architecture.md の内容確認
# 実装対象の特定
# レビュー指針確認（docs/ 配下をgrep）

# 2. Phase 1: コア機能の骨組み作成
# クラス・ファイルのガワ作成
# 空メソッド定義
# マイグレーション作成・実行（必要な場合）
# git add . && git commit -m "コア機能の骨組み作成"

# 3. Phase 2: コア機能の実装
# メソッド1の実装とテスト作成・実行
# git add . && git commit -m "メソッド1実装"
# メソッド2の実装とテスト作成・実行
# git add . && git commit -m "メソッド2実装"
# （コア機能のメソッドを順次実装、各完了時にコミット）

# 4. Phase 3: サポート機能の実装
# サポート機能の骨組み作成
# サポート機能メソッドの実装とテスト作成・実行
# 各完了時にコミット

# 5. Phase 4: 統合テスト
# 統合テスト作成・実行
# git add . && git commit -m "統合テスト追加"

# 6. 最終確認
# 設計通りの動作確認
# 全テスト結果確認
```

## チェックポイント

**段階的実装の確認:**
- [ ] Phase 1: コア機能の骨組みが作成され、コミットされているか
- [ ] Phase 2: コア機能のメソッドが一つずつ実装され、各完了時にコミットされているか
- [ ] Phase 3: サポート機能の骨組みと実装が完了し、各完了時にコミットされているか
- [ ] Phase 4: 統合テストが作成・実行され、コミットされているか

**実装品質の確認:**
- [ ] task-architecture.mdの設計内容が全て実装されているか
- [ ] 設計で定義されたファイル・クラス・メソッドが作成されているか
- [ ] データベース変更が設計通り実行されているか
- [ ] レビュー指針ガイドライン（docs/配下）に従っているか
- [ ] 設計で指定されたテストが実装され通過しているか
- [ ] 設計通りの動作が確認できているか
- [ ] 設計に対する追加や変更を行っていないか

**コミット履歴の確認:**
- [ ] 各Phase完了時に適切なコミットメッセージでコミットされているか
- [ ] 実装の進捗が分かりやすいコミット履歴になっているか
